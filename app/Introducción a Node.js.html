Introducción a Node.js

Node.js es un lenguaje de código abierto que se ejecuta utilizando el motor V8. Dado que es de código abierto, Node.js es compatible con Linux, Windows y Mac OSX. El motor V8 es una tecnología de alto rendimiento de código abierto desarrollada por Google para su navegador Google Chrome. JavaScript es ampliamente utilizado por los desarrolladores en la funcionalidad del lado del cliente. Node.js ocupa el lugar del servidor utilizando el mismo lenguaje.

Node.js se basa en eventos y utiliza E/S asíncronas y no bloqueantes. Con JavaScript del lado del servidor, las aplicaciones de Node procesan y enrutan las solicitudes de servicios web desde el cliente. En el paso 1, el usuario selecciona una opción en la interfaz de usuario, escrita en HTML y CSS. En el paso 2, esta acción del usuario activa el código JavaScript que implementa la lógica empresarial en el lado del cliente, por ejemplo, la validación de entradas.

En el paso 3, la aplicación JavaScript realiza una llamada a un servicio web a través de HTTP con una carga útil de datos JSON. El servicio web REST, que forma parte de una aplicación Node.js que se ejecuta en el servidor Node, recibe la solicitud HTTP. En el paso 4, el servicio web REST procesa la solicitud y devuelve el resultado al cliente como una carga útil JSON a través de HTTP. Aunque los desarrolladores pueden seguir utilizando JavaScript para la funcionalidad del navegador en marcos como AngularJS, Dojo y jQuery, ahora pueden utilizar Node.js en los mismos componentes de la arquitectura en los que utilizan Java, Perl, C++, Python y Ruby. Empresas como Uber y Yahoo! utilizan Node.js en producción, así como LinkedIn, GoDaddy, eBay y PayPal. Node.js se basa en eventos y utiliza E/S asíncronas y no bloqueantes. Express.js es un marco altamente configurable para crear aplicaciones en Node.js.

Antes de crear tu primera aplicación de Node.js, familiarízate con el entorno de desarrollo integrado (IDE) y algunos conceptos clave de Node.js. Express.js simplifica el proceso de desarrollo de aplicaciones en Node.js y ofrece funciones que permiten crear tu aplicación de manera eficiente. Estas funciones incluyen la gestión de activos públicos como imágenes, archivos CSS y JavaScript, la capacidad de renderizar plantillas HTML en el servidor y la definición de rutas para aceptar y procesar solicitudes de clientes. Además, aprenderás sobre archivos importantes como server.js y package.json, que contienen información relevante para tu proyecto, incluyendo dependencias y scripts.

import() y require()!

En Node.js, los módulos representan archivos que contienen código JavaScript enfocado y encapsulado, diseñado con un propósito específico. Estos módulos pueden ser individuales o una colección de archivos y carpetas relacionados. Los desarrolladores confían en los módulos debido a su capacidad de reutilización y su eficacia para dividir el código complejo en partes más manejables.

  1. Cuando una aplicación externa necesita utilizar el código de un módulo, se requiere que la aplicación llame a ese módulo.
  2. La aplicación externa puede llamar al módulo mediante una declaración de importación () o require (). La elección de la declaración depende de la especificación del módulo.
  3. Los módulos agrupados en un directorio se denominan paquetes, y las especificaciones de módulo definen las convenciones y estándares que rigen la creación de paquetes en JavaScript para aplicaciones de Node.js.
  4. Las especificaciones más comunes en las aplicaciones de Node.js son los módulos CommonJS y ES.

En Node.js, por defecto, el código JavaScript se trata como módulo CommonJS. No obstante, los autores de bibliotecas pueden habilitar fácilmente los módulos ES en un paquete Node.js con solo cambiar la extensión de archivo del paquete.

Los módulos CommonJS utilizan la declaración require () para importar otros módulos, mientras que los módulos ES utilizan la función import ().

Para usar un módulo fuera de su archivo de origen, es necesario exportarlo. En el caso de los módulos CommonJS, se utiliza la declaración module.exports. Por otro lado, los módulos ES emplean la palabra clave "export" para lograrlo.

Las diferencias entre require e import también se manifiestan en el código de la aplicación:

  • La declaración require puede ubicarse en cualquier lugar del archivo, lo que significa que se puede utilizar en sentencias y funciones condicionales. Sin embargo, la instrucción import debe colocarse al comienzo del archivo.
  • La sentencia require está vinculada dinámicamente, lo que implica que los errores en la vinculación entre la definición y la llamada a la función solo se identifican durante la ejecución. Por otro lado, las instrucciones de importación están vinculadas estáticamente, lo que significa que los errores de vinculación se detectan en tiempo de compilación.
  • Los módulos requeridos son sincrónicos, lo que implica que se cargan y procesan secuencialmente. Los módulos importados con la importación son asincrónicos y pueden procesarse simultáneamente.

Estas diferencias en el enfoque entre require e import pueden afectar la eficiencia en aplicaciones a gran escala que involucran la carga de numerosos módulos.

Para ilustrar los procesos de exportación e importación, aquí hay ejemplos de código:

Exportación desde un módulo CommonJS:


                const message = "Hello programmers";
                module.exports = message;

Importación de un módulo CommonJS:


                const msg = require('./message.js');
                console.log(msg);

Exportación desde un módulo ES:


                const myValue = 1;
                export { myValue };

Importación de un módulo ES:


                import { myValue } from './module.mjs';
                console.log(myValue);

Node.js es un marco de programación en el lado del servidor que utiliza JavaScript como lenguaje de programación. Lo que lo hace especialmente atractivo es que muchos desarrolladores ya están familiarizados con el lenguaje JavaScript. Además, se destaca por su enfoque en la programación simultánea y su naturaleza ligera.

Node.js opera como un entorno de aplicaciones de un solo subproceso que gestiona las operaciones de entrada/salida (E/S) mediante eventos en lugar de bloquear operaciones de E/S de manera síncrona. En lugar de bloquear operaciones de E/S asíncronas, se escriben funciones de devolución de llamada para gestionar los resultados una vez finalizados. Esto lo convierte en una opción ideal para desarrolladores que buscan crear aplicaciones de servidor escalables y simultáneas utilizando funciones como las funciones de devolución de llamada y el bucle de eventos de ejecución de Node.js.

En Node.js, cada archivo JavaScript es un módulo, y un módulo se corresponde con un archivo de script. Puede crear y extender sus aplicaciones utilizando los numerosos módulos de utilidad que ofrece el motor de ejecución de Node.js. Por ejemplo, el módulo HTTP de Node.js permite desarrollar aplicaciones que escuchan las solicitudes HTTP y responden con mensajes de respuesta HTTP.

Para instanciar un servidor web en Node.js, se utiliza la función http.createServer. El servidor web se almacena en una variable llamada "servidor". Esta función createServer puede incluir una función de devolución de llamada opcional que gestiona las solicitudes entrantes y envía respuestas adecuadas. A continuación, se puede configurar el servidor para escuchar en un puerto específico utilizando la función http.listen, por ejemplo, para configurar el servidor para escuchar en el puerto 8080.
