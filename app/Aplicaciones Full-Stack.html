Desarrollo de Aplicaciones Full-Stack con Node.js

Cuando hablamos de aplicaciones full-stack, nos referimos a:

  • El lado del cliente:

    • Esto incluye el sitio web y la aplicación móvil que los usuarios utilizan.

  • El lado del Servidor:

    • Este se encarga de procesar las solicitudes del lado del cliente y enviar respuestas. En la actualidad, la nube alberga el servidor web, el servidor de aplicaciones y la base de datos.

Open Source y Plataforma Cruzada

Javascript es una elección ideal en el lado del cliente para validar las páginas HTML. Dado su facilidad de uso, Javascript se ha extendido para ser utilizado en la programación del lado del servidor, lo que nos lleva a Node.js. Node.js es un entorno de ejecución.

Motor V8

Cualquier código que escribas necesita ser procesado y convertido a una forma legible por la máquina. El código Javascript de Node.js utiliza el motor V8 de Google. V8 es un motor de alto rendimiento de código abierto desarrollado por Google. Todos los navegadores Google Chrome vienen con el motor V8, y otros navegadores modernos como Microsoft Edge y Firefox también utilizan el motor V8 para Node.js.

Eventos, Asincronía, No Bloqueante, Monohilo

Los procesos en un servidor pueden ser "monohilo" o "multihilo". Monohilo significa que solo se procesa un comando a la vez. Aunque Node.js es monohilo, es asincrónico y no bloqueante. Esto significa que mientras se ejecuta un proceso, el programa no tiene que esperar a que el proceso termine. Node.js es impulsado por eventos. Cuando Node.js realiza una operación de entrada/salida (E/S), como leer desde la red, acceder a una base de datos o al sistema de archivos, se desencadena un evento. En lugar de bloquear el hilo y desperdiciar el tiempo del procesador esperando, Node.js reanuda las operaciones cuando llega la respuesta o, en otras palabras, cuando se produce el evento de respuesta. Durante ese tiempo, el servidor no está bloqueado y puede realizar otras tareas, lo que le da una apariencia de ser multihilo.

Carga JSON

JSON significa Notación de Objetos de Javascript. El JSON está formateado como "pares clave-valor". La carga es la información transmitida entre el cliente y el servidor. Cuando el cliente necesita enviar datos al servidor, lo hace en forma de un objeto JSON, como se muestra en el ejemplo:

        {
            "nombre": "Juan",
            "edad": "24",
            "correo": "juanparker@gmail.com"
        }

El objeto anterior es un JSON. Cuando estos datos se envían como parte de una solicitud, los valores se pueden extraer simplemente utilizando, por ejemplo, request.nombre.

La respuesta también se envía como JSON. Aquí tienes un ejemplo:

        {
            "estado": "ok",
            "mensaje": "Agregado exitosamente"
        }

Framework Express

Si bien Node.js tiene paquetes para crear un servidor, el framework Express facilita la creación de APIs y puntos de acceso. Un punto de acceso API es el punto de entrada específico para una solicitud del cliente al servidor.
